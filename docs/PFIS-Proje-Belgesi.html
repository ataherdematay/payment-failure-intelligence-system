01
Projenin Amacı ve Motivasyonu Neden bu sistem inşa edildi?

PFIS (Payment Failure Intelligence System), online ödeme işlemlerindeki başarısızlıkları analiz eden, sınıflandıran ve optimize eden bir yapay zeka destekli karar destek sistemidir.

🎯
Temel Problem: Ödeme orkestrasyon platformları binlerce işlemi saniyeler içinde farklı sanal POS sağlayıcılardan geçirir. Her başarısız işlem hem müşteri deneyimini hem de işletme gelirini doğrudan etkiler. Ancak pek çok firmada bu başarısızlıkların neden gerçekleştiği bilinmez, gateway'lerin birbirine göre performansı ölçülmez ve önlem alınması reaktiftir.

Bu sistem, ödeme verilerini canlı olarak izleyen, başarısızlık nedenlerini makine öğrenmesiyle tahmin eden, otomatik içgörüler üreten ve iş kararlarını simüle eden eksiksiz bir platform sunmaktadır.

04
Geliştirme Aşamaları Projenin chronolojik inşa süreci
Faz 1 — Temel Altyapı
Veri Modeli ve Veritabanı
Ödeme işlemlerini temsil eden veri şeması tasarlandı. transactions tablosu; işlem tutarı, ödeme yöntemi, ülke, cihaz, gateway, başarı/başarısızlık durumu ve başarısızlık nedeni alanlarını içermektedir.
Faz 2 — Backend API
NestJS REST API ve Analitik Modüller
NestJS çatısıyla Clean Architecture prensiplerine uygun servis katmanları kuruldu. Transactions, Analytics, Insights ve ML proxy modülleri yazıldı. Swagger UI entegre edildi (/api/docs). TypeORM ile otomatik şema senkronizasyonu sağlandı.
Faz 3 — ML Servisi
RandomForest Başarısızlık Sınıflandırıcı
FastAPI üzerinde bağımsız bir ML microservice inşa edildi. scikit-learn ile RandomForestClassifier eğitildi; 5 başarısızlık sınıfı için tahmin yapılmaktadır. /predict, /model/metrics ve /model/retrain endpoint'leri hazırlandı.
Faz 4 — Frontend Dashboard
5 Sayfalık Gerçek Zamanlı Dashboard
Next.js App Router ile 5 sayfalı dashboard tamamlandı. TanStack Query ile tüm API çağrıları cache edildi. Recharts entegrasyonuyla area, bar, pie ve composed chart'lar eklendi. Glassmorphism temalı koyu renk tasarım sistemi oluşturuldu.
Faz 5 — Craftgate Entegrasyonu
Sektöre Özel Hikaye ve Preset Senaryolar
Simulation sayfasına Craftgate ürünlerine doğrudan atıfla 4 preset senaryo eklendi: "Drop Square Gateway", "Retry Autopilot ON", "Peak Season", "Current State".
Faz 6 — Production Deployment
Vercel + Railway Üzerinde Tam Canlı Sistem
Frontend Vercel'e, Backend ve ML servisi Railway'e deploy edildi. Railway PostgreSQL veritabanı otomatik şema senkronizasyonu (synchronize: true) ve startup seed mekanizmasıyla 5.000 gerçekçi işlem kaydı ile dolduruldu. ESM/CJS uyumsuzluğu (uuid → crypto.randomUUID()) giderildi; port binding sorunları çözüldü.
Faz 7 — Login & Admin Paneli
JWT Auth + CSV Veri Yükleme + Şirket Verisi Desteği
POST /auth/login endpoint'i ve JWT tabanlı kimlik doğrulama eklendi. Admin panelinde şirketin kendi gerçek ödeme verilerini CSV formatında yükleyip analiz edebileceği alan hayata geçirildi. ML modelinin API üzerinden yeniden eğitilmesi (POST /model/retrain) ve veri sıfırlama butonları eklendi. Sidebar'a giriş/çıkış ve admin linki entegre edildi.

Backend API Referansı NestJS (Railway) ve FastAPI ML Servisi (Railway)

NestJS Backend — /api/v1

Endpoint Method Auth Açıklama
/auth/login POST — JWT token döner (email + password)
/transactions GET — Filtrelenebilir, sayfalandırılmış işlem listesi
/transactions/summary GET — KPI özeti: toplam, başarısız, oran, risk
/transactions/upload POST JWT CSV dosyası yükle — şirketin gerçek verisi
/transactions/seed POST — 5.000 sentetik demo verisi ekle
/transactions/clear DELETE JWT Tüm işlem kayıtlarını sil (admin)
/analytics/failure-by-reason GET — Neden bazlı başarısızlık dağılımı
/analytics/daily-trend GET — Son N günlük işlem trendi
/analytics/gateway-performance GET — Gateway bazlı başarısızlık oranı ve hacim
/analytics/hourly-pattern GET — 24 saatlik saat bazlı desen
/insights GET — Otomatik üretilen içgörü kartları
/ml/predict POST — FastAPI ML servisine proxy tahmin isteği

FastAPI ML Service

Endpoint Method Açıklama
/health GET Model yüklü mü, servis sağlıklı mı?
/predict POST Başarısızlık nedeni tahmini + güven skoru
/model/metrics GET Doğruluk, F1, sınıf bazlı precision/recall
/model/retrain POST Modeli yeniden eğit (güncel DB verisine göre)

Craftgate ile İlişkisi Bu projenin Craftgate'in iş problemleriyle örtüşmesi

Craftgate, Türkiye'nin lider ödeme orkestrasyon platformudur. 500+ üye işyeri, %99.99 uptime ve 2025'te 1.19 milyar ₺ kurtarılan gelir ile ödeme altyapısında kritik bir konum tutmaktadır. PFIS'in tasarımı bu gerçek dünya problemleri dikkate alınarak yapılmıştır:

Craftgate Ürünü PFIS Karşılığı Değer
Autopilot & Ödeme Tekrar Deneme Simulation → "Retry Autopilot ON" preset'i Retry başarı oranının gelir kurtarıma etkisini modellemek
Akıllı ve Dinamik Ödeme Yönlendirme Analytics → Gateway Performans grafiği Hangi gateway'in yüksek başarısızlık ürettiğini görmek; yönlendirme kararı vermek
Gelişmiş Üye İşyeri Paneli Overview + Analytics sayfaları İşletmenin ödeme sağlığını gerçek zamanlı izlemek
Fraud & Risk Değerlendirme ML Predictions + Insights kritik uyarıları Şüpheli işlemleri fatura edilmeden önce sınıflandırmak
🔀
Örnek İş Senaryosu — "Drop Square Gateway":
Analytics sayfasında Square gateway'inin %39.1 başarısızlık oranıyla diğer gateway'lerin (%25-29) çok üzerinde olduğu tespit edilir. Insights sayfası bunu WARNING kart olarak raporlar. Simulation sayfasında "Drop Square Gateway" preset'i seçildiğinde başarısızlık oranının %31 → %24.1'e, kayıp gelirin $465K → $360K'ya düştüğü anlık hesaplanır. Bu, tam olarak Craftgate'in Akıllı Routing ürününün müşterilere kazandırdığı değerdir.

Teknik Beceri Örtüşmesi

  • Backend: NestJS ile modüler, SOLID prensipli REST API — Craftgate backend stack'ine (Java/Spring) analoji kurulabilir temiz mimari
  • Frontend: Next.js + TanStack Query + TypeScript — Craftgate Frontend Developer ilanında istenen stack ile birebir örtüşme
  • DevOps Kültürü: Backend/frontend ayrımı yok; tam-yığın geliştirici yaklaşımı — Craftgate'in ürün ekibi modeli
  • Domain Bilgisi: Ödeme terminolojisi (gateway, retry, fraud, chargeback), POS entegrasyonu, başarısızlık sınıflandırması
  • Veri Odaklı: Analitik sorgular, ML pipeline, insight üretimi — veriyi karar desteğine dönüştürme
